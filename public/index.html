3. Informações do Sistema

3.1 Identificação do Projeto

3.2 Funcionalidades Implementadas

  • Sistema completo de gerenciamento de usuários (CRUD)
  • Sistema completo de gerenciamento de disciplinas (CRUD)
  • Dashboard com indicadores estatísticos em tempo real
  • Validações de segurança e integridade de dados
  • Criptografia de senhas de acesso
  • Banco de dados relacional PostgreSQL
  • Controle de versão com Git e GitHub
  • Testes automatizados com Selenium WebDriver

3.3 Arquitetura da API REST

Módulo de Usuários:

  • POST /api/usuarios - Cadastro de novo usuário
  • GET /api/usuarios - Listagem de usuários cadastrados
  • GET /api/usuarios/:id - Consulta de usuário específico
  • PUT /api/usuarios/:id - Atualização de dados de usuário
  • DELETE /api/usuarios/:id - Remoção de usuário do sistema

Módulo de Disciplinas:

  • POST /api/disciplinas - Cadastro de nova disciplina
  • GET /api/disciplinas - Listagem de disciplinas cadastradas
  • GET /api/disciplinas/:id - Consulta de disciplina específica
  • PUT /api/disciplinas/:id - Atualização de dados de disciplina
  • DELETE /api/disciplinas/:id - Remoção de disciplina do sistema
